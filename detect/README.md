# 阶段二: 装甲板检测与中心点输出

## 目标

使用阶段一训练导出的 ONNX 模型，在 C++ 中完成装甲板检测推理，输出每帧中所有装甲板（最多 4 个）的中心点坐标。

## 你需要修改的文件

| 文件 | 说明 |
|------|------|
| `src/detector.cpp` | 实现模型加载、前处理、推理、后处理 |
| `src/target_selector.cpp` | 从检测结果中提取中心点填入 FrameResult |
| `src/visualizer.cpp` | 可视化绘制 |
| `src/main.cpp` | 初始化检测器、打开输入源、可视化显示 |

不要修改 `src/judge.cpp`、`src/evaluate.cpp`、`include/judge.h`、`include/evaluate.h`。

### 大二额外任务: EKF 预测

在完成上述中心点输出的基础上，实现 EKF 对目标运动的预测。

- `include/ekf_predictor.h` 和 `src/ekf_predictor.cpp` 为空壳，自行设计与实现
- 需要自行将 EKF 模块接入 `CMakeLists.txt`（添加源文件、链接依赖库等）
- 在画面中可视化检测点与预测点的区别即可

**目标要求:**

- **平移预测（必须完成）**: 对目标中心点的平移运动进行完整的 EKF 建模与预测，能够在画面中看到预测点领先于检测点
- **旋转建模（尽力完成）**: 对目标的旋转/角度变化进行建模，重点在于你对原理的理解和如何将其应用到实际场景中，验收时需要能解释你的设计思路

## 评估标准

综合得分 = 精度得分 × 70% + 速度得分 × 30%，满分 100。

### 精度得分 (0-100)

对每帧中的每个 ground truth 中心点，在你的输出中找最近的匹配点，通过高斯衰减将距离误差转换为分数:

```
单点得分 = 100 × exp(-dist² / (2σ²))    σ = 15px
```

| 误差 (px) | 单点得分 |
|-----------|----------|
| 0         | 100      |
| 5         | 95       |
| 10        | 80       |
| 15 (σ)    | 61       |
| 20        | 41       |
| 30 (2σ)   | 13       |
| 45 (3σ)   | 1        |

每帧得分 = 该帧所有 gt 点得分的平均。未匹配到的 gt 点得 0 分。  
精度得分 = 所有帧得分的平均。

### 速度得分 (0-100)

```
速度得分 = min(FPS / 30, 1.0) × 100
```

30 FPS 及以上为满分，低于 30 FPS 线性衰减。

> 速度评估的测试环境为 **Jetson Orin Nano 8GB (CUDA 架构)**，最终验收时统一在该平台上运行计时。
